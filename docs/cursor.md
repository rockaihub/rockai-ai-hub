# Cursor 使用指南

Cursor 是一款面向开发者的 AI 编程工具，适合在真实项目中进行代码补全、文件理解、错误排查、重构辅助和功能开发。相比通用聊天工具，Cursor 更贴近开发环境，可以围绕项目文件进行上下文协作。

## 工具简介

Cursor 可以看作带有 AI 能力的代码编辑器。它不仅能回答代码问题，还可以结合当前项目上下文，帮助你理解代码结构、修改文件、生成测试和解释报错。

常见用法包括：

- 阅读陌生项目
- 解释函数和模块关系
- 生成业务代码
- 修复报错和类型问题
- 重构重复逻辑
- 编写单元测试
- 生成接口文档或注释

## 适合场景

### 项目理解

接手一个新项目时，可以让 Cursor 帮你解释目录结构、关键模块和入口文件。

```text
请帮我阅读这个项目，说明前端入口、路由结构、状态管理和接口请求分别在哪里。
```

### 功能开发

Cursor 适合辅助实现明确的小功能。你需要给出需求、边界条件和预期行为。

### 错误排查

遇到编译错误、类型错误或测试失败时，可以把报错信息和相关文件一起交给 Cursor 分析。

### 重构和测试

Cursor 可以帮助提取函数、减少重复代码、补充测试用例，但你仍需要运行测试并审查变更。

## 使用建议

### 1. 需求要具体

不要只说“帮我改这个项目”。更好的方式是说明目标、文件范围和验收标准。

```text
请在用户设置页增加邮箱校验逻辑：
1. 邮箱为空时提示“请输入邮箱”
2. 格式错误时提示“邮箱格式不正确”
3. 保持现有 UI 风格不变
4. 修改后补充相关测试
```

### 2. 让 AI 先读代码

在复杂项目里，先让 Cursor 总结现有模式，再要求它动手改代码，会更稳。

### 3. 必须运行测试

AI 生成的代码可能能看但不能跑。每次修改后都应该运行编译、测试或至少启动项目检查关键页面。

### 4. 不要把敏感代码随意外发

如果项目包含密钥、客户数据或内部逻辑，应先了解团队政策和工具的数据处理方式。

## 常见问题

### Cursor 适合编程新手吗？

适合，但不要只复制代码。建议让它解释每一步思路，并自己运行、调试和理解。

### Cursor 可以替代开发者吗？

不能。它是效率工具，可以减少重复劳动和辅助排错，但需求判断、架构设计、质量控制仍需要开发者负责。

### Cursor 和 ChatGPT 写代码有什么区别？

ChatGPT 更像通用对话助手；Cursor 更贴近编辑器和项目上下文，适合直接在代码库里协作。

### 订阅前应该考虑什么？

建议看你是否高频写代码、是否需要项目级上下文能力、是否能接受在编辑器内与 AI 协作，以及团队是否允许使用相关工具。
